### Modulo 2- levantamento de requisitos (continuaçaõ)
     - Oque é levantamento de requisito 
        - Transformar as necessidades do negocio em informações técnicas 
     - Requisitos funcionais e não funcionas 
        - RF - o que o sistema vai fazer 
        - RNF- como o sistema vai fazer
    - Requisitos, regras, restrições 
        - requisitos: a parte técnica do sistema 
        -regras: as necessidades dos clientes 
        - restrições: limitações impostas pela, linguagem, leis, normas 
    - Modelos de documentação de requisitos
    |campo|conteudo|exemplo|
    |---|---|---|
    |ID|codigo|Rf-001|
    |titulo|valor|tela de login|
    |descrição|o ator deve ser capaz de| o cliente conseguirá 
    fazer o login|
    |prioridade| valor de prioridade|alta|
    |regras vinculadas|outras regras ralacionadas| Rn-001- Banco de dados|
### Tecnicas de levatamento do requisito
        - Briefing
        - Brainstorm
        - questionario
        - entrevista 
        - etnografia 
        - workshop 
        - prototipagem
### Brienfing
        - coleta de dados inicial
        - não existe formato pré-definido
        - permite coletas de diversas informações do cliente
          - manual de identidade visual(cores, logo, slogan, fontes...)
          -concorrentes 
          -prazos
          -orçamentos
          -identidade da empresa (missão, visão, valores)
          -público-alvo
### Brainstorm
        -reunião criativa com objetivo de solucionar um ou mais "problemas"
        -podem ser estruturas ou não-estruturados
        -reunião curta com foco bem definido
        -ferramentas
         -miro
         -mentimeter
         -onenote
### questionario
        - perguntas direcionadas ao cliente, com objetivo de montar o espaço do projeto 
        (documentação final do projeto) 
        - formas de fazer o questionario
          - reunião com pergunta diretas (qualitativas)
          - formulario de coleta de dados (quatitativas)
### entrevistas
         -instrumentos de coleta de dados com o usuario do projeto 
         -entrevista podem ser formais e informais
         -nessa etapa é necessario um MVP (produto mínimo viável)
### etnografia
        - observação da cultura de empresa, a forma como ela atende e se relaciona com o seu cliente publico interno ou externo
### workshop
        -tecnica de levatamento de requisitos que acontecem na apresentação de um MVP, para um grupo seleto de pesssoas (stakeholders) com intuito de absover a reação do publico ao produto/projeto
        -eventos, feiras e workshop de produto especificos
### prototipagem
       - Criação de um elemento de desing para simulação de um projeto, com intuito de testar funcionalidades e elementos graficos antes de começar a escrever codigos 
       -os protótipos podem ser de 
        - baixa fidelidade (rafe): rascunho de um desenho, pode ser feito a mão 
        -media fidelidade (rough): desenhos no computador para marcação de espaços e layouts, não precisa de textos nem cores 
        -alta fidelidades; prototipos focado nas aplicações de funcionalidade iguais aos dos projetos que serão desenvolvidos posteriomente em código
